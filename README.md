# Todo-list
Приложение является простой реализацией системы управления задачами, включающей все CRUD-операции. В качестве базы данных для хранения задач используется **PostgreSQL**, сам сервис реализован на Go с использованием фреймворка **Fiber**.

## Запуск
_0. необходимо наличие docker, docker-compose_
1. На основе `.env.example` создать свой `.env`, содержащий конфигурацию подключения к базе данных
2. Собрать и запустить контейнер
```bash
docker compose up --build
```
4. по умолчанию сервис слушает на порту `:3000`
```bash
http://localhost:3000
```

## API
все запросы к API используют базовый url `/api/tasks`
#### GET
1. `/` - получить все задачи
2. `/:id` - получить конкретную задачу
3. `/priority/:priority` - получить задачи с конкретным приоритетов (от 0 до 2, где 0 - наименьший)

#### POST
1. `/` - добавить задачу

полный `body` для создания задачи:
```json
{
  "title": "task-title",
  "description": "",
  "priority": 0
  "deadline": "2026-03-20T14:22:05Z"
}
```
обязательное только поле `title`, остальные можно не указывать

#### PUT
1. `/:id` - обновление задачи. можно записывать в `body` только те поля, которые нужно изменить (dto выше)

#### PATCH
1. `/:id/toggle` - изменение статуса задачи (выполнено/нет)

#### DELETE
1. `/:id` - удаление задачи
